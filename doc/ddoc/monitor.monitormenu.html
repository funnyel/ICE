class MonitorMenu [abstract] ¶
Base class for monitor menus.
MonitorMenu implementations are usually generated by generate_monitor_menu(). They provide access to SubMonitors of some subsystem through their menu items.

string generate_monitor_menu(string name, string[] monitor_names, string[] monitor_classes) ¶
Generate a MonitorMenu implementation class providing access to specified monitors.
Used with string mixins.

If the monitored class is templated, syntax "Monitored$T$U" can be used. In that case, both the monitor menu and monitors accessed will be templated with specified template types.

Note:

To use this as a mixin, you have to import gui.guimenu .

Parameters:

nameName of the monitored class. Generated class will be named "name ~ Monitor".
monitor_namesMenu item texts corresponding to respective monitors.
monitor_classesNames of monitor classes accessed through the menu.

Returns:

Generated menu monitor implementation code.

final package class ExampleMonitorableMonitor(T) : MonitorMenu 
{     
    private ExampleMonitorable!(T) monitored_;
    public this(ExampleMonitorable!(T) monitored)
    {
        auto factory = new GUIMenuHorizontalFactory;
        factory.add_item("Back", &back_to_parent);
        factory.add_item("MonitorTextA", &MonitorTextA);
        factory.add_item("MonitorTextB", &MonitorTextB);
        super(factory);
        monitored_ = monitored;
    }
    private:
        void MonitorTextA(){set_monitor.emit(new MonitorClassA!(T)(monitored_));}
        void MonitorTextB(){set_monitor.emit(new MonitorClassB!(T)(monitored_));} 
}
