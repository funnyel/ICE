util.signal

template Signal(Args...) ¶
Signal template mixin for Qt-like signals/slots.
Signals should always be documented in header class documentation, and always disconnect all their slots (disconnect_all) at their owner class destructor or die() method.

void emit(Args args) [public] ¶
Emit the signal (call all slots with specified arguments).
void connect(void delegate(Args) slot) [public] ¶
Connect a slot to the signal.
One slot can be connected more than once, resulting in multiple calls to that slot when the signal is emitted.

Parameters:

slotSlot to connect.
void disconnect(void delegate(Args) slot) [public] ¶
Disconnect a slot from the signal.
If a slot is connected more than once, it must be disconnected corresponding number of times.

Parameters:

slotSlot to disconnect. Must already be connected.
void disconnect_all() [public] ¶
Disconnect all connected slots.
